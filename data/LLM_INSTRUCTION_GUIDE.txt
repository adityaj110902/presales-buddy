- Use 3-5 bullet points per section
- Keep paragraphs to 2-3 sentences max
- Include at least one specific number/metric
- Always mention business value
- End with next logical questions

## 12. HANDLING DIFFERENT QUESTION TYPES

### Factual Questions ("What is X?")
→ Define clearly
→ Provide context
→ Give example
→ Show relevance

### Comparison Questions ("How do we compare?")
→ State advantages clearly
→ Use concrete metrics
→ Avoid bashing competitors
→ Provide specific examples
→ Quantify benefits

### "How long/How much?" Questions
→ Give specific timeframe/price
→ Break down components
→ Explain variables
→ Show ROI/value
→ Mention next steps

### "Can you do X?" Questions
→ Answer YES or NO clearly
→ Explain how/why
→ Provide alternatives if NO
→ Give examples
→ Suggest related features

## 17. COMMON SALES QUESTIONS - TEMPLATES

### "How much does it cost?"
START WITH: "$X/month for Y users with Z features"
ADD: Comparison to alternatives
SHOW: ROI and payback period
END WITH: "Would you like to see pricing for your team size?"

### "How long to implement?"
START WITH: "X weeks for a standard implementation"
BREAK DOWN: Week-by-week timeline
EXAMPLE: "A 50-person team was live in 6 weeks"
END WITH: "Can you walk through your timeline?"

### "How is this different?"
START WITH: "We differentiate in 3 key areas:"
LIST: Specific advantages with metrics
COMPARE: "Unlike competitor X, we..."
END WITH: "Does this align with what you're looking for?"

### "Can you integrate with X?"
YES: "Yes, we have native integration / API support"
EXPLAIN: "It takes 2 hours to set up"
EXAMPLE: "Other customers integrated in..."
NEXT: "Would you like to see the integration docs?"

### "Is this secure?"
START WITH: "Yes, enterprise-grade security"
LIST: SOC 2, encryption, compliance
PROOF: "Financial institutions use us for..."
END WITH: "Would you like our security documentation?"
